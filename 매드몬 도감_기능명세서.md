## 1. 프로젝트 개요
---
### 한 줄 정의
매드노바 도감: 몰입캠프 참가자 상호 평가 및 팀 빌딩 플랫폼

### 제품 목표
몰입캠프 참가자들이 서로의 역량과 특성을 효과적으로 파악하고, 재미있는 상호 평가 및 게임화된 경험을 통해 팀 빌딩 과정을 지원하며, 최종적으로 효율적인 협업 환경을 조성하는 가치를 제공합니다.

### 배경
기존 몰입캠프 환경에서 참가자들은 팀 구성 시 서로의 강점과 약점을 파악하는 데 어려움을 겪을 수 있으며, 이는 비효율적인 팀 형성으로 이어질 수 있습니다. '매드노바 도감'은 이러한 문제를 해결하고 참가자 간의 네트워킹 및 협업 시너지를 극대화하기 위해 기획되었습니다.

### 사용자 문제
몰입캠프 참가자들은 서로의 역량을 정확히 파악하기 어렵고, 이로 인해 비효율적인 팀 구성이 발생하며, 캠프 참여 몰입도가 저하될 수 있습니다.

### 해결 방안
'매드노바 도감'은 참가자 간 상호 평가 시스템을 제공하여 개인의 역량을 다각도로 시각화하고, AI 기반의 팀 조합 분석 기능을 통해 효율적인 팀 빌딩을 지원합니다. 또한, 카드 도감 형태로 정보를 제공하고 칭호 시스템을 도입하여 캠프 참여에 대한 게임화된 즐거움을 선사합니다.

### 차별점
단순한 팀원 평가를 넘어, 6가지 핵심 역량에 대한 상세 평가와 '매드노바' 카드 도감 형태의 시각화, 그리고 최신 평가 점수를 추측하기 어렵게 만든 독특한 점수 산정 로직을 제공합니다. 특히, AI 기반의 팀 시너지 분석 및 맞춤형 카드 질문 기능은 참가자들이 서로를 깊이 이해하고 최적의 팀을 구성하는 데 실질적인 도움을 주며, 기존 캠프 내 평가 시스템과는 차별화된 참여형 경험을 제공합니다.

### 타겟 사용자
몰입캠프에 참여하는 모든 참가자 및 운영진. 특히, 새로운 팀원과의 협업을 중시하고, 다양한 팀 구성 실험을 통해 자신의 역량을 극대화하고 싶어 하는 적극적인 참가자를 주요 대상으로 합니다. 또한, 팀원들의 역량을 객관적으로 파악하고 효율적인 팀 구성을 지원하려는 운영진도 포함됩니다.

### 사용자 시나리오
몰입캠프 참가자 '김개발'은 캠프 첫날 '매드노바 도감'에 로그인하여 기본 비밀번호를 변경하고 자신의 프로필 사진, 초기 능력치(가상 값), 한 줄 자기소개를 설정합니다. 이후 팀을 생성하거나 기존 팀에 코드를 입력하여 참여합니다. 프로젝트가 끝난 후, 김개발은 팀원들을 '공격력', '방어력' 등 6가지 항목으로 평가하고 적절한 칭호를 부여합니다. 김개발은 카드 도감에서 다른 참가자들의 카드를 조회하며, '프론트엔드 장인' 칭호를 받은 '이디자'의 카드에 AI에게 '이디자와 함께 작업하면 어떤 시너지를 낼 수 있을까요?'라고 질문하여 잠재적 팀 조합에 대한 통찰을 얻습니다.

### 핵심 지표
평가 참여율 (전체 참가자 중 평가를 완료한 비율), 팀 생성 및 참여율, AI 카드 질문 기능 사용 횟수, 서비스 재방문율, 사용자 만족도 설문조사 결과 (NPS), 카드 도감 조회수

### 리스크
평가의 공정성 및 객관성 확보의 어려움 (악의적인 평가 방지), 개인 정보 및 데이터 보안 문제, AI 답변의 품질 및 정확성 부족으로 인한 사용자 불만, 낮은 참여율로 인한 서비스 활성화 실패, 기술적 문제로 인한 서비스 지연 및 오류 발생 가능성

사용자는 새로운 팀을 생성하고 고유한 팀 참여 코드를 발급받을 수 있습니다. 다른 사용자는 이 코드를 입력하여 팀에 참여할 수 있습니다.

**수용 기준:**
```
1. ○ 사용자는 팀을 생성하고 고유한 팀 참여 코드를 얻을 수 있다.
2. ○ 다른 사용자는 팀 참여 코드를 입력하여 해당 팀에 가입할 수 있다.
3. ○ 한 사용자는 여러 팀에 속할 수 있다.

## 4. 매드노바 카드 도감 조회
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

모든 참가자는 다른 참가자들의 매드노바 카드를 조회할 수 있습니다. 카드는 프로필 사진, 이름, 칭호, 육각형 그래프(앞면), 상세 육각형 그래프, 받은 칭호 전체 투표 수, 한 줄 자기소개(뒷면)를 포함합니다.

```
1. ○ 사용자는 다른 참가자의 매드노바 카드를 조회할 수 있다.

### 4.6 평가 완료자만 카드 상세 열람
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

도감 카드의 상세 정보(능력치, 칭호, AI 분석 등)는 사용자가 ‘활성화된 동료 평가’를 모두 완료한 경우에만 열람할 수 있다. 평가가 미완료이면 카드 상세 영역은 블러 처리하고 자물쇠(잠금) 오버레이를 표시하여 내용을 확인할 수 없게 한다.

- ‘활성화된 동료 평가’ 기준: 현재 참여 중인 팀/프로젝트 중 평가 기간이 열려있거나(활성) 제출이 요구되는 평가 항목
- 잠금 상태에서도 카드 목록/썸네일/앞면 요약은 노출 가능(도감 탐색은 허용)
- 잠금 안내 문구: “동료 평가를 완료하면 상세 정보를 볼 수 있어요”
- 잠금 해제 CTA: “평가하러 가기” 버튼으로 평가 화면으로 이동
- 평가를 모두 완료하면 즉시(또는 재진입 시) 잠금 해제 반영

예외/정책:
- 본인 카드(내 카드)도 동일 정책 적용(참여 유도 목적)
- 평가 대상자가 없는 경우(예: 팀 미참여)에는 잠금 없이 열람 허용 또는 안내 후 평가 참여 유도(정책 선택 필요)

추가 고려:
- 잠금 상태에서 AI 질문 기능도 비활성화(버튼 disabled + 잠금 툴팁)
- 접근성: 블러만으로 구분하지 않고 자물쇠 아이콘/텍스트 제공


#### 4.6.1 평가 완료 여부 판정(게이팅)

사용자의 ‘활성화된 동료 평가’가 모두 제출 완료인지 판정한다.

- 입력: 사용자, 현재 참여 팀/프로젝트, 활성 평가 항목 목록
- 출력: isUnlocked(boolean), remainingCount(남은 평가 수), lockedReason(문구 키)
- 판정 규칙(초안):
  - 활성 평가 항목이 1개 이상이고 그중 미제출이 존재하면 잠금
  - 활성 평가 항목이 없으면 잠금 해제(기본)
- 상태 갱신:
  - 앱 진입/도감 진입 시 재계산
  - 평가 제출 성공 시 즉시 재계산(실시간 반영)

로그/지표:
- 잠금 노출 횟수, 잠금 → 해제 전환율, 잠금 상태에서 ‘평가하러 가기’ 클릭률
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중


#### 4.6.2 잠금 UI(블러 + 자물쇠 오버레이)

카드 상세 화면(뒷면/상세 정보 영역)에 블러 처리와 자물쇠 오버레이를 적용해 내용을 확인할 수 없게 한다.

- 적용 범위: 능력치/세부 설명/칭호 상세/AI 분석 요약 등 ‘상세’로 분류된 영역
- 노출 허용: 카드 목록, 카드 썸네일, 카드 앞면의 최소 요약(이름/대표 칭호 정도)
- 문구:
  - 타이틀: “잠겨 있어요”
  - 안내: “동료 평가를 완료하면 상세 정보를 볼 수 있어요”
  - 보조: “남은 평가 {remainingCount}개” (가능 시)
- 동작:
  - 상세 영역 탭/스크롤은 가능하되 내용은 항상 블러
  - 자물쇠 영역 탭 시 안내 모달 또는 토스트 표시

접근성:
- 아이콘 + 텍스트 병행(블러만으로 의미 전달 금지)
- 스크린리더용 대체 텍스트 제공
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중


#### 4.6.3 잠금 해제 유도(‘평가하러 가기’ CTA)

잠금 상태에서 사용자가 바로 평가로 이동할 수 있게 CTA를 제공한다.

- CTA 버튼: “평가하러 가기”
- 이동 대상:
  - 기본: 사용자의 활성 평가 목록 화면
  - 가능하면: 미제출 평가가 있는 팀/프로젝트의 평가 화면으로 바로가기(우선순위 규칙 필요)
- 완료 후:
  - 평가 제출 성공 → 도감 상세로 돌아오면 잠금 해제 상태 반영
  - 잔여 평가가 있으면 잠금 유지 + 남은 개수 업데이트

예외:
- 네트워크 오류/제출 실패 시: 실패 안내 + 재시도
- 활성 평가가 없는데 잠금 화면에 들어온 경우(상태 불일치): 재계산 후 즉시 해제
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중


#### 4.6.4 AI 질문 기능 잠금 연동

잠금 상태에서는 카드 AI 질문 관련 입력/버튼을 비활성화한다.

- 대상: [기능] 개별 카드 AI 질문, 팀 조합 분석 질문, 카드 비교 분석 질문, AI 분석 결과 제공의 진입/버튼
- UI:
  - 버튼 disabled
  - 툴팁/안내: “동료 평가를 완료하면 AI 분석을 사용할 수 있어요”
- 잠금 해제 시 즉시 활성화
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중

## 5. AI 기반 카드 분석 및 질문
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

사용자는 원하는 카드를 선택하여 AI에게 해당 카드에 대한 질문을 할 수 있습니다. 또한, 임의의 팀을 선택하여 조합이나 상성에 대한 질문, 두 카드를 비교하는 질문도 가능합니다.

**수용 기준:**
```
1. ○ 사용자는 개별 카드에 대해 AI에게 질문을 할 수 있다.
2. ○ 사용자는 임의의 팀을 구성하여 AI에게 팀 조합 및 상성에 대한 분석 질문을 할 수 있다.
3. ○ 사용자는 두 개 이상의 카드를 선택하여 AI에게 비교 분석 질문을 할 수 있다.
4. ○ AI는 질문에 대한 분석 결과를 제공해야 한다.
```


### 5.1 개별 카드 AI 질문

사용자가 특정 카드를 선택하여 AI에게 해당 참가자의 역량, 특성, 강점 등에 대한 자유로운 질문을 할 수 있는 기능입니다.


#### 5.1.1 카드 선택 UI

사용자가 원하는 카드를 선택할 수 있는 UI를 제공합니다. 카드 도감에서 특정 카드를 클릭하면 AI 질문 페이지로 이동하거나, AI 질문 페이지에서 카드를 검색/선택할 수 있습니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 사용자 | **기기**: Web


#### 5.1.2 개별 카드 정보 전달

선택된 카드의 정보(사용자명, 능력치, 칭호, 자기소개 등)를 AI 질문 시스템으로 전달합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 사용자 | **기기**: Web


#### 5.1.3 AI 질문 입력 필드

사용자가 선택한 카드에 대해 자유로운 형식의 질문을 입력할 수 있는 텍스트 입력 필드를 제공합니다(예: '이 사람과 함께 작업하면 어떤 시너지를 낼 수 있을까요?').

사용자가 입력한 질문이 공백이 아닌지, 일정 길이 이상인지 검증합니다. 유효하지 않은 질문은 제출을 거부합니다.

#### 5.3.2 선택된 카드 정보 전달

비교 대상으로 선택된 모든 카드의 정보(사용자명, 능력치, 칭호, 자기소개 등)를 AI 질문 시스템으로 전달합니다.

사용자가 선택된 카드들을 비교 분석하는 질문을 입력할 수 있습니다(예: '이 두 사람의 강점과 약점을 비교해주세요' 또는 '이 세 사람과 함께라면 어떤 프로젝트를 성공시킬 수 있을까요?').

### 5.4 AI 분석 결과 제공
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 사용자 | **기기**: Web

AI가 사용자의 질문에 대해 데이터 기반의 통찰력 있는 분석 결과를 생성하고 사용자에게 제공하는 기능입니다.


#### 5.4.1 AI API 연동

선택된 카드/팀 정보와 사용자 질문을 외부 AI API(예: OpenAI API, Claude API)로 전송하는 기능을 구현합니다. 요청 포맷과 응답 처리를 정의합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 사용자 | **기기**: Web


#### 5.4.2 AI 질문 프롬프트 생성

사용자의 질문과 카드/팀 정보를 기반으로 AI에게 전달할 상세한 프롬프트를 구성합니다. 프롬프트에는 카드 정보, 사용자 질문, 응답 형식 등이 포함됩니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 사용자 | **기기**: Web


#### 5.4.3 AI 응답 수신 및 처리

AI API로부터 분석 결과를 수신하고, 응답이 유효한 형식인지 검증합니다. 오류 발생 시 적절한 에러 핸들링을 수행합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 사용자 | **기기**: Web


#### 5.4.4 AI 분석 결과 UI 표시

AI가 제공한 분석 결과를 사용자 친화적인 형식으로 화면에 표시합니다. 텍스트, 불릿 포인트, 구분된 섹션 등을 활용하여 가독성을 높입니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 사용자 | **기기**: Web


#### 5.4.5 로딩 및 대기 상태 표시

AI가 분석을 진행 중일 때 로딩 스피너 또는 진행 상태 메시지를 표시하여 사용자가 응답 대기 중임을 인식할 수 있도록 합니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: 사용자 | **기기**: Web


#### 5.4.6 질문 및 답변 기록 저장

사용자의 AI 질문과 AI의 응답을 데이터베이스에 저장합니다. 나중에 사용자가 이전 분석 결과를 다시 확인할 수 있도록 합니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: 사용자 | **기기**: Web


#### 5.4.7 API 레이트 리미팅

사용자의 AI 질문 요청 빈도를 제한하여 API 비용 및 과부하를 방지합니다. 일정 시간 동안의 요청 제한(예: 분당 5회)을 설정합니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: 사용자 | **기기**: Web
