# MadMon — 디자인 지시 프롬프트 (Claude Code용)

아래 지시를 그대로 따라 MadMon의 전체 UI를 구현하라.

---

## 1. 세계관 & 컨셉

MadMon은 팀원 상호 평가 서비스지만, 절대 "평가 툴"처럼 보이면 안 된다.
**"개발자라는 별들을 관측하고 기록하는 미래의 심우주 관측소(Deep-Sky Observatory)"**라는 세계관을 가진 하나의 작품으로 만든다.

- 참가자 1명 = 별(Star). 각자 다른 크기·색·밝기·반짝임을 가진다.
- 팀 = 은하(Galaxy). 팀원들은 리스트가 아니라 우주 공간에 산개한 별로 존재한다.
- 평가 = 관측(Observation). 사람을 점수 매기는 게 아니라 별을 스캔·분석해 잠재력을 발견하는 행위다.
- 첫인상 목표: "와… 이거 진짜 예쁘다. 우주를 탐험하는 게임 같다."

모든 UI 문구는 이 세계관의 언어를 쓴다: OBSERVE, TARGET LOCK, SCANNING, SPECTRAL ANALYSIS, CONSTELLATION FRAGMENTS(칭호), OBSERVATIONS(평가 횟수), STARS DETECTED.

## 2. 절대 금지 (Anti-patterns)

- 프로필 카드 / 명함형 UI / 그리드 카드 목록
- Material Design, Bootstrap 느낌, 평범한 대시보드·기업 관리페이지
- 촌스러운 네온 사이버펑크, 과도한 글래스모피즘
- 이모지, 좌측 보더 액센트 박스, 과한 그라데이션 남용
- 일반 차트처럼 보이는 레이더 차트 (별자리처럼 재해석할 것)

## 3. 색상

배경 (깊은 우주 — 완전한 검정 금지, 짙은 남색 계열):
- `#020617` (최심부 베이스)
- `#081126` (중간)
- `#0B1736` (상층)
- 기본 배경: `linear-gradient(160deg, #020617 0%, #081126 55%, #0B1736 100%)`

포인트 컬러:
- 하늘색 `#7DD3FC` (주 액센트: HUD 강조, 링크, 차트 선)
- 청록 `#5EEAD4` (시스템 활성 상태: TARGET LOCK, ONLINE, 버튼 그라데이션)
- 보라 `#A78BFA` (보조 액센트: 위성, 성운)
- 별빛 흰색 `#eef4ff` / `#f2f6ff` (제목, 별 코어)
- 본문 텍스트 `#c6d6ef` / 보조 `#a9bcd9` / 흐린 라벨 `#64789c` / 최저 대비 `#3d4f70`

별 색 변주: `#9ed2ff`(청백), `#f2f6ff`(순백), `#c4b0ff`(보라)
주 버튼: `linear-gradient(120deg, #7DD3FC, #5EEAD4)` + 글자색 `#020617`
성운: 보라 `rgba(96,78,200,.15)`, 청 `rgba(56,120,220,.13)`, 청록 `rgba(64,190,180,.08)` radial-gradient + blur(30px)
테두리: `1px solid rgba(125,180,255,.16)` — 항상 얇고 은은하게.

## 4. 타이포그래피

- 영문 디스플레이/로고: **Space Grotesk** (300–600). 로고는 letter-spacing 5–10px.
- 한국어 본문: **Noto Sans KR** (300–500 위주, 가늘게).
- HUD/시스템 라벨: **IBM Plex Mono**, 9–11px, letter-spacing 2–4px, 대문자.
- 언어 규칙: 라벨·시스템 문구는 영어(mono, 대문자), 설명·본문은 한국어.
- 1920 기준 본문 13px 이상, 흐린 라벨도 9px 미만 금지.

## 5. 배경 연출 (모든 화면 공통)

1. 남색 그라데이션 베이스
2. 성운 blob 2–3개: radial-gradient + blur, 26–38초 주기로 아주 느리게 표류
3. 작은 입자 별 ~70개: 1–3px, 개별 랜덤 주기(2.5–7.5s)의 twinkle(opacity .25↔1)
4. 별 사이 희미한 별자리 점선: `rgba(140,175,235,.14)`, dasharray
5. 애니메이션은 절대 사용자를 방해하지 않는 수준 — 느리고, 부드럽고, 은은하게.

## 6. 핵심 화면 & 인터랙션

### 인트로 (첫 진입, 스킵 가능)
검은 우주 중앙에 맥동하는 별 하나 → "MADNOVA / DEEP-SKY OBSERVATORY" → 얇은 진행바 → "INITIALIZING OBSERVATION SYSTEMS…" 깜빡임 → 1초 페이드아웃. 총 ~3초, 클릭 시 즉시 스킵.

### 메인 은하 화면
- 풀블리드 우주. UI는 네 모서리의 HUD 텍스트뿐 (좌상: 로고, 우상: GALAXY TEAM-XX, 좌하: "N STARS DETECTED · 조작 힌트", 우하: RA/DEC 좌표 · ZOOM 배율 · 실시간 시계).
- 팀원 = 산개한 별. 크기 12–21px 코어 + 다층 box-shadow 글로우 + 개별 주기 breathe 애니메이션.
- 호버: 글로우 확산(halo radial-gradient fade-in) + 별 아래 지정 코드(`MDM-011 α`)와 이름 등장.
- 자유 탐험: 드래그 팬, 휠 줌(커서 기준, 0.55–2.8x), 더블클릭 리셋. 탐험 중 transition 없음.

### 별 선택 → 프로필
- 클릭: 청록 점선 조준원 + 펄스 링 + "TARGET LOCK" (~0.75초) → 카메라가 1.4초 cubic-bezier로 별을 화면 좌측 1/3 지점까지 딥줌(scale 2). 다른 별은 opacity .22로 감쇠.
- 우측에서 홀로그램 패널 슬라이드 인: 반투명 남색 그라데이션 + backdrop-blur + 청록 코너 브래킷. **카드가 아니라 관측 계기판**.
- 패널 내용: 원형 프로필(주위를 도는 점선 궤도 + 위성 점 2개) / 영문코드·역할 / 이름 / ✦ 대표 칭호 / 자기소개 / SPECTRAL ANALYSIS(능력치 별자리) / CONSTELLATION FRAGMENTS(획득 칭호 조각) / OBSERVATIONS 횟수 / OBSERVE 버튼.
- "◂ RETURN TO GALAXY"로 복귀.

### 능력치 = 별자리 차트
육각 레이더를 별자리로 재해석: 흐린 육각 그리드(3겹) + 꼭짓점은 빛나는 별(popIn), 데이터 선은 **하나씩 순차적으로 그려지는** 애니메이션(stroke-dashoffset, 0.22s 간격), 내부 채움 `rgba(125,211,252,.09)`. 축 라벨은 영문 mono + 한국어 작게.
6개 특성: ATTACK 공격력 / DEFENSE 방어력 / CREATIVITY 창의성 / RESOLVE 문제 해결력 / VELOCITY 속도 / SYNC 협업.

### 평가 = 관측
- OBSERVE 클릭 → 화면 dim → 별 주위 회전 점선 링 + 펄스 → "SCANNING..." → "ANALYZING STAR..." → "SPECTRAL PROFILE ACQUIRED" (~2.9초).
- 관측 모드: 특성별 10칸 세그먼트 게이지(클릭 입력, 채워진 칸은 하늘색→청록 그라데이션 + 글로우), 실시간 별자리 프리뷰, 전 항목 입력 시에만 "⌁ TRANSMIT RECORD · 기록 전송" 활성화.
- 전송 → "✦ OBSERVATION LOGGED / 관측 기록이 저장되었습니다" → 프로필 복귀, 관측 횟수 +1.

### 칭호
배지 금지. 대표 칭호는 이름 옆 ✦와 글로우 텍스트로 가장 크게. 획득 칭호는 "✦—" 별자리 조각 모양의 얇은 보더 칩. 프로필 이미지 궤도를 도는 위성 점으로도 암시.

### AI 채팅 (MADNOVA CORE)
우주선 관제 컴퓨터 콘솔 스타일. 좌측 패널, 헤더 "◈ MADNOVA CORE / OBSERVATORY AI · ONLINE". 메시지는 말풍선이 아니라 로그: "CORE ⌁"(청록) / "YOU ▸"(하늘색) 라벨 + 본문. 응답 대기 시 "⌁ TRANSMITTING…" 깜빡임. AI 말투: 차분한 관제 시스템 톤 한국어, 별·스펙트럼·궤도 은유, 2–4문장.

## 7. 모션 원칙

- 등장: fadeIn / fadeUp(14px) 0.6–1.2s, 카메라: 1.2–1.4s `cubic-bezier(.25,.9,.25,1)`
- 패널: 옆에서 60–70px 슬라이드 + 페이드, 0.8–0.9s
- 반복 모션(반짝임·공전·펄스)은 전부 다른 주기로 — 동기화된 움직임 금지
- 회전 궤도: 6–22s linear infinite. 빠른 회전 금지.
- 모든 것이 "고급스럽고 자연스럽게". 튀는 바운스, 강한 flash 금지.

## 8. UI 원칙

- 여백을 아낌없이. 화면의 대부분은 우주 그 자체다.
- 가독성 최우선 — 글로우는 텍스트 뒤에, 텍스트 위에 올리지 않는다.
- 게임 UI처럼 화려하되 복잡하지 않게. 한 화면에 동시에 열리는 패널은 최대 2개.
- 모서리 radius는 2–4px로 절제 (둥근 카드 감성 금지). 원형은 별·궤도·조준원에만.
- 세련되고 프리미엄한 톤 유지. 사용자는 "우주를 탐험"해야 한다.
